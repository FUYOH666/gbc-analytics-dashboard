# Security policy

## Supported versions

This is a **demo / portfolio** project, not a commercial product with LTS branches. Security fixes apply to the **default branch** (`main` or `master`) going forward.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for undisclosed security vulnerabilities — that can expose other users before a fix is available.

**Preferred options:**

1. Use [GitHub private vulnerability reporting](https://github.com/FUYOH666/gbc-analytics-dashboard/security/advisories/new) if enabled for the repository.

2. Or contact the maintainer via the channels listed in [README.md](README.md) (e.g. website or Telegram), with a clear subject line such as `[Security] gbc-analytics-dashboard`.

## What to include

- Description of the issue and affected component (route, dependency, deployment).
- Steps to reproduce or proof-of-concept **without** posting real production secrets.
- Suggested fix if you have one (optional).

## What not to post publicly

- API keys, bot tokens, `service_role` keys, or `CRON_SECRET` values.
- Private internal network URLs (VPN, office networks).
- Customer or production data from live systems.

We aim to acknowledge reports within a reasonable timeframe; this is a side project, so response times may vary.
